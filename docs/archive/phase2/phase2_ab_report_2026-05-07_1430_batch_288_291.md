## Why still 0 submittable

The 3 PROV all fail on **CONCENTRATED_WEIGHT**. CW is a structural
property of the alpha — even with perfect sharpe + fitness, if positions
concentrate on a few names BRAIN rejects. Common causes:

1. **`ts_zscore`/`ts_rank` on small-coverage fields** — only stocks with
   IV/derivative data participate; weight concentrates on options-rich
   names (10-20% of TOP3000)
2. **Long history fields like `composite_factor_score_derivative`** —
   only liquid names have enough history → narrow universe → CW
3. **Lacking explicit cross-sectional dispersion** — no `rank()` or
   `group_neutralize()` wrapper to spread weight across industries

Of the 3 alpha:
- pk=6602: `multiply(-1, ts_av_diff(high, 60))` — uses `high` (universal),
  CW likely from sub-universe coverage gaps rather than field selectivity
- pk=6603: `ts_av_diff(implied_volatility_put_90, 5)` — IV class CW
- pk=6604: `ts_rank(implied_volatility_put_30, 20)` — IV class CW

## Why "0 submittable" persists despite all fixes

The fix chain solved the **labeling**: alpha that aren't BRAIN-submittable
are now correctly NOT labeled PASS. But labeling改 does NOT make alpha
submittable — that requires generating alpha that:
- pass our internal gate (sh≥1.25, fit≥0.95)
- AND pass BRAIN's full check matrix (CW, sub-universe, self-correlation,
  pyramid eligibility)

Current LLM-generated alpha distribution doesn't have intersection of
both. The IV-class generates high sharpe but structural CW; the broad
universe ts_op alpha don't reach our sharpe bar.
