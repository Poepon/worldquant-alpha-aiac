## Why 0 submittable alpha — root cause stack

The chronic "0 can_submit" symptom was a 3-layer bug:

1. **Fix D bug** — `evaluate_with_brain_checks` treated empty BRAIN checks
   as "approve". 5/8 mining-time bcs=true were false positives.
2. **P0 gate misalignment** — internal gate 0.8/0.5 was探索 bar; BRAIN
   submit bar is ~1.25/1.0. Even ignoring Fix D bug, 80% of "PASS" had
   sharpe < BRAIN's bar.
3. **Fix C absence** — when BRAIN actually said FAIL at mining time
   (rare, given Fix D bug masked it), no PASS-downgrade pathway existed.

All three fixes now in master. Next batch will demonstrate true behavior.
